## What's Built

### Backend packages (in `backend/`)

| Package | Purpose |
|---------|---------|
| `config/` | Environment-based configuration via godotenv |
| `db/` | PostgreSQL connection pool (pgxpool), Goose migrations (embedded), sqlc queries + generated code |
| `session/` | Redis-backed session store, context helpers |
| `middleware/` | Request ID, structured logging, CORS, recovery, auth (RequireAuth, RequireRole, RequirePlatformAdmin), body limit, API key auth, rate limiting |
| `service/` | All business logic: auth, player, team, org, venue, league, tournament, division, registration, season, pod, announcement, match, scoring_preset, match_series, bracket, court_queue, standings, overlay, source_profile, dashboard, search, activity_log, api_key, upload |
| `handler/` | HTTP handlers for all services, response helpers, error mapping |
| `engine/` | Stateless pickleball scoring engine (side-out + rally, game/match detection, serve rotation, timeouts) |
| `bracket/` | Bracket generation (single-elim, double-elim, round-robin) + progression logic |
| `overlay/` | Canonical overlay data contract, 6 themes, resolver, webhook HMAC validation, external API poller |
| `pubsub/` | Redis pub/sub wrapper with 6 channel types |
| `ws/` | WebSocket handler (gorilla/websocket) for 6 real-time channels |
| `jobs/` | Background cleanup job for expired quick matches |
| `router/` | Chi v5 router assembly with all route groups |
| `testutil/` | Test database helper, test server wiring |

## Known Deviations from Spec

These are accepted deviations documented during reviews:

1. **4E schema diverged from plan** — Different column names (`team1_id` not `team_1_id`), `series_format TEXT` instead of `series_config JSONB` (later added via migration 00022). Accepted and patched.
2. **Standings are team-only** — `player_id`/`registration_id` dropped from standings_entries. Individual player standings unsupported.
3. **Rate limiter is in-memory** — Not Redis-backed as planned. Works for single-instance; needs Redis for multi-instance.
4. **External API surface is minimal** — Only `/health` exposed. Tournament/league/match read endpoints not yet built.
5. **API key scoping is flat array** — Plan had hierarchical scope+scope_id. Simpler model shipped.
6. **Pool-play/pool-to-bracket bracket formats** — Not yet implemented in BracketService.
